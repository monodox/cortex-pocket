# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Security Features

### Privacy by Design
- **100% Local Processing**: All AI computations happen on-device
- **No Network Communication**: Zero data transmission to external servers
- **Encrypted Storage**: Chat history encrypted with AES-256
- **Secure Memory**: Sensitive data cleared from memory after use

### Data Protection
- **Local Storage Only**: All data remains on user's device
- **No Telemetry**: No usage analytics or tracking
- **No Cloud Sync**: No automatic data synchronization
- **User Control**: Complete user control over data retention

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

### How to Report
1. **Email**: Send details to security@cortex-ai.dev
2. **Include**: 
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact assessment
   - Suggested fix (if available)

### Response Timeline
- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Resolution**: Within 30 days for critical issues

### What to Expect
- Acknowledgment of your report
- Regular updates on investigation progress
- Credit in security advisories (if desired)
- Responsible disclosure coordination

## Security Best Practices

### For Users
- Keep the app updated to latest version
- Use device lock screen protection
- Enable device encryption
- Review app permissions regularly

### For Developers
- Follow secure coding practices
- Regular dependency updates
- Code review for security issues
- Static analysis integration

## Threat Model

### In Scope
- Local data protection
- Memory safety
- Input validation
- Dependency vulnerabilities

### Out of Scope
- Physical device access
- Operating system vulnerabilities
- Hardware-level attacks
- Social engineering
